                   设备引导程序(Bootloader_0)版本说明 ---源程序			      
                          >=64K  引导的起始地址  0x8000800                         
						  
说明：	1、本引导适用于STM32L15X中等容量的芯片;
		2、本引导程序如若分区表改变(DevPartTab.c、DevPartTab.h)，请必须修改(main.h)中的宏DEVICE_PARTION_TABLE_ADDR
		   的地址,保持与分区表同步,否则设备会出现异常;
		3、对于特殊标记的说明
  *********************************************************************************************************
		  标记名称       		*			            标识说明 
  *********************************************************************************************************
	bootloader_0_flag    		*  若为0xAA788755则直接跳转到App区,否则跳转到bootloader1;
	<BOOTLOADER_0_JUMP_FLAG>
  *********************************************************************************************************
		4、此跳转函数void JumpToCode(unsigned int addr)没有判断,用户区或者bootloader1的堆栈初始化地址检测
	//	if (((*p) & 0x2FFE0000 ) == 0x20000000)	//判断用户代码的的堆栈地址落在:0x20000000~0x2001ffff区间内
	//	{
	//		__set_MSP(*p);		//取出初始的MSP值
	//		((void (*)(void))(*(p + 1)))();
	//	}
		主要是因为,我们升级采用先写固件再改升级跳转标记,若升级固件失败,而标记不会被擦除,倘若判断用户代码的堆栈
		是否初始化,也不能保证设备能够正常运行,因此在此处没有判断;
